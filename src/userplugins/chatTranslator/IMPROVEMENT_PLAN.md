# ChatTranslator Improvement Plan

이 문서는 Equicord `messageTranslate`의 단순한 사용 흐름을 참고해, `ChatTranslator`를 더 쓰기 쉽고 안전하게 다듬기 위한 작업 계획입니다.

## 참고한 messageTranslate 특징

- Google Translate를 이용해 받은 메시지를 자동 번역한다.
- 원문 언어를 자동 감지하고, 신뢰도가 낮으면 번역하지 않는다.
- 번역된 메시지에는 짧은 `Translated` 표시를 붙인다.
- 이미 번역 표시가 붙은 메시지는 다시 번역하지 않는다.
- 설정이 적어서 켜자마자 이해하기 쉽다.

## 진행 원칙

- 기존 고급 기능은 유지하되, 기본 경험은 더 조용하고 단순하게 만든다.
- UI 문구는 가능한 한 짧고 읽기 쉽게 바꾼다.
- 새 옵션은 기존 사용자 설정을 최대한 깨지 않게 기본값을 정한다.
- 각 단계마다 빌드와 최소 정적 검사를 통과시킨다.

## 1. 언어 코드 대신 언어 이름 표시

상태: 완료

### 목표

번역 표시에서 `ja`, `ko`, `en` 같은 코드가 그대로 보이지 않도록 하고, 사용자가 바로 이해할 수 있는 언어 이름을 표시한다.

### 작업

- 번역 결과의 `sourceLanguage` 표시값을 한 번 더 정규화한다. ✅
- Google / DeepL / Azure 감지 결과가 코드로 들어와도 `Japanese`, `Korean`처럼 보여준다. ✅
- 빠른 연결 테스트나 실패 문구에도 같은 표시 함수를 재사용한다. ✅

### 완료 기준

- 메시지 액세서리에서 `Translated from Japanese`처럼 표시된다.
- 알 수 없는 코드는 기존 코드값을 그대로 보여준다.

## 2. Simple Mode 추가

상태: 완료

### 목표

공식 `messageTranslate`처럼 켜면 조용히 번역만 보여주는 간단 모드를 제공한다.

### 작업

- 설정에 `simpleMode`를 추가한다. ✅
- Simple Mode에서는 기본 액션을 줄인다. ✅
  - 접기/펼치기, 재번역, 원문 토글 같은 버튼을 숨긴다. ✅
  - 짧은 `Translated` 또는 `Translated from ...` 표시만 남긴다. ✅
- 실패/건너뜀 이유는 기본적으로 숨기고, 수동 번역 때만 보여주는 방향을 검토한다.

### 완료 기준

- Simple Mode를 켜면 메시지 표시가 공식 플러그인처럼 단순해진다.
- 기존 고급 표시 모드는 그대로 유지된다.

## 3. 기본 번역 언어 설정 단순화

상태: 완료

### 목표

받은 메시지/보낸 메시지/채널별 언어 설정이 많아 처음 쓰는 사람이 헷갈리지 않도록 한다.

### 작업

- 설정 화면 상단에 `Default Target Language` 개념을 추가한다. ✅
- 받은 메시지 출력 언어와 보낸 메시지 출력 언어를 기본값 하나로 맞출 수 있게 한다. ✅
- 별도 언어 설정은 고급 설정처럼 보이게 정리한다. ✅

### 완료 기준

- 처음 설정하는 사용자가 기본 번역 언어 하나만 보고도 사용할 수 있다.
- 고급 사용자는 기존처럼 받은/보낸/채널별 언어를 따로 설정할 수 있다.

## 4. Google 신뢰도 설정을 프리셋으로 개선

상태: 완료

### 목표

숫자 입력만으로는 이해하기 어려운 감지 신뢰도 설정을 쉬운 선택지로 바꾼다.

### 작업

- `Loose`, `Balanced`, `Strict`, `Custom` 프리셋을 제공한다. ✅
- 각 프리셋 설명을 추가한다. ✅
  - Loose: 더 많이 번역하지만 오탐 가능성 증가
  - Balanced: 기본 추천값
  - Strict: 틀린 감지를 더 적극적으로 피함
- 기존 숫자 값은 유지하고, 프리셋 UI가 그 값을 조정하게 한다. ✅

### 완료 기준

- 사용자가 숫자 의미를 몰라도 신뢰도 수준을 고를 수 있다.
- 기존 `googleConfidenceRequirement` 설정과 호환된다.

## 5. 이미 번역된 메시지 재번역 방지 강화

상태: 완료

### 목표

다른 번역 플러그인이나 사용자가 붙인 번역 표시를 다시 번역하지 않도록 한다.

### 작업

- 자동 번역 전에 이미 번역된 메시지인지 검사한다. ✅
- 다음 패턴을 우선 검토한다. ✅
  - `(Translated)`
  - `*(Translated)*`
  - `translated from ...`
  - `Translated from ...`
- 옵션으로 끄고 켤 수 있게 할지 검토한다. ✅ 옵션으로 추가

### 완료 기준

- 흔한 번역 표시가 붙은 메시지는 자동 번역을 건너뛴다.
- 수동 번역은 가능하게 유지할지 정책을 정한다.

## 6. 실패 이유 표시 기본 경험 정리

상태: 완료

### 목표

공식 플러그인처럼 실패했을 때 채팅을 지저분하게 만들지 않고, 필요한 상황에서만 이유를 보여준다.

### 작업

- `Show Skip / Failure Reasons` 기본값과 동작을 재검토한다. ✅
- 자동 번역 실패는 조용히 원문만 두는 방향으로 조정한다. ✅
- 수동 번역 실패는 이유를 보여준다. ✅
- 개발/디버깅용 상세 표시 옵션은 유지한다. ✅

### 완료 기준

- 일반 사용자는 실패 메시지를 자주 보지 않는다.
- 문제를 확인하고 싶은 사용자는 설정으로 상세 이유를 볼 수 있다.

## 7. 메시지 내용 직접 교체 모드 검토 및 실험 옵션 추가

상태: 검토 완료, 구현 보류

### 목표

공식 `messageTranslate`처럼 메시지 본문 자체를 번역문으로 바꾸는 방식을 실험 옵션으로 제공할 수 있는지 검토한다.

### 주의점

이 방식은 UI가 깔끔하지만 충돌 위험이 있다.

- 원문 접근이 어려워질 수 있다.
- 다른 메시지 관련 플러그인과 충돌할 수 있다.
- 복사/검색/액세서리 표시와 동작이 달라질 수 있다.

### 작업

- 기존 액세서리 방식과 직접 교체 방식의 충돌 가능성을 조사한다. ✅
- 실험 옵션 이름을 정한다.
  - 예: `Experimental: Replace message text directly`
- 켰을 때 액세서리 표시를 어떻게 줄일지 정한다. 보류

### 완료 기준

- 안전하게 구현 가능하면 실험 옵션으로 추가한다.
- 충돌 위험이 크면 구현하지 않고 조사 결과를 문서에 남긴다.

## 7번 검토 결과

상태: 검토 완료, 구현 보류

공식 `messageTranslate`처럼 메시지 본문을 직접 교체하는 방식은 현재 `ChatTranslator` 구조에는 바로 넣지 않는 편이 안전합니다.

### 보류 이유

- `ChatTranslator`는 받은 메시지 번역을 메시지 액세서리로 표시하고, 원문/번역 토글, 접기, 재번역, 무시 상태, 캐시 상태를 모두 이 액세서리 흐름에 묶어 둔다.
- 본문을 직접 교체하면 원문 표시, `Translated + Original`, `Toggle Original / Translation` 모드와 역할이 겹친다.
- 다른 메시지 관련 플러그인과 같은 렌더링 위치를 건드릴 가능성이 커진다.
- 보낸 메시지 자동 번역 기능과 받은 메시지 본문 교체 기능이 동시에 켜질 때 사용자가 실제 원문/번역 상태를 헷갈릴 수 있다.

### 대신 적용한 방향

- 2번 `Simple Mode`로 공식 플러그인처럼 조용하고 단순한 표시 경험을 제공한다.
- 원문 접근과 고급 액션은 기존 액세서리 방식에 남긴다.
- 직접 교체 방식은 나중에 별도 브랜치에서 충돌 테스트를 충분히 한 뒤 실험 옵션으로 다시 검토한다.
